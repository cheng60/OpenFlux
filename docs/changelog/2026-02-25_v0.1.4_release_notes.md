# OpenFlux v0.1.4 Release Notes

**Created:** 2026-02-25  
**Author:** Development Team  
**Status:** Published  

## Overview

OpenFlux v0.1.4 fixes scheduled task artifact detection, Router bind popup blocking, Lark multimedia preview, and Embedding model config persistence issues. Also improves LLM config UI layout.

---

## Bug Fixes

### 1. Fix scheduled task artifact detection missing

**Impact:** Scheduled task core functionality  
**Trigger:** After scheduled task execution, Agent successfully generates files but the artifact panel cannot detect or display them  
**Root cause:** `executeScheduledAgent` doesn't broadcast `chat.progress` WebSocket events (real-time detection path broken), and doesn't extract/save artifacts after completion (persistence path broken)  
**Fix:**
- Broadcast `chat.progress` events in `onToolStart`/`onToolCall` callbacks
- New `extractAndSaveScheduledArtifacts` function to extract file artifacts from tool call records after task completion
- Support detecting files from `filesystem.write/copy/info`, `process/opencode`, with stdout regex fallback

**Modified:** `gateway/src/gateway/standalone.ts`

---

### 2. Fix Router bind popup blocking other views

**Impact:** UI interaction  
**Trigger:** When Router is unbound, the bind code input popup stays on top, blocking settings and scheduler views  
**Fix:** Hide Router bind UI when switching to settings/scheduler views, restore when returning to chat

**Modified:** `src/main.ts`

---

### 3. Fix Lark multimedia message preview

**Impact:** Router multimedia functionality  
**Trigger:** Images/files sent from Lark cannot be previewed in client chat interface  
**Fix:** Implement multimedia message bubble rendering and click-to-preview, integrated with existing file preview mechanism

**Modified:** `src/main.ts`, `src/styles/main.css`

---

### 4. Fix Embedding model config persistence & change detection

**Impact:** Long-term memory functionality  
**Trigger:** Switching Embedding model and restarting loses config; same-dimension model switch doesn't trigger vector rebuild  
**Fix:**
- Embedding model config correctly saved to `server-config.json` and restored on restart
- `MemoryConfig` adds `embeddingModel` field, persisted to `memory_meta` table
- Change detection no longer relies solely on dimensions; model name change also triggers rebuild

**Modified:** `gateway/src/gateway/standalone.ts`, `gateway/src/agent/memory/manager.ts`, `gateway/src/agent/memory/types.ts`

---

## Improvements

### 5. LLM config source switch UI relocation

**Impact:** Settings panel  
**Description:** Moved LLM config source switch UI (local/managed) from top of settings panel to within the "Server" tab before "Model Configuration" section

**Modified:** `src/main.ts`

---

## Upgrade Notes

- Direct overlay installation, no additional configuration needed
- Existing configs are not affected
- Embedding model config will auto-migrate on first startup
