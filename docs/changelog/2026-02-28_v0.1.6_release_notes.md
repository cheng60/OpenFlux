# OpenFlux v0.1.6 Release Notes

**Created:** 2026-02-28  
**Author:** Development Team  
**Status:** Published  

## Overview

OpenFlux v0.1.6 focuses on **internationalization (i18n)** and **Agent core engine refactoring**. Adds full English/Chinese bilingual support, refactors Agent Loop history management and execution logic, and optimizes multiple frontend interactions.

---

## New Features

### 1. Internationalization (i18n) system

**Impact:** Global  
**Description:** Full English/Chinese bilingual support covering frontend UI and backend Agent system prompts.
- New `src/i18n/` module: `index.ts` (core engine), `zh.ts` (Chinese pack, 454 entries), `en.ts` (English pack, 453 entries)
- Supports `data-i18n`, `data-i18n-placeholder`, `data-i18n-title`, `data-i18n-html` DOM bindings
- Auto-detects browser language preference, supports manual switch with localStorage persistence
- `locale-changed` custom event for dynamic UI refresh

**New files:** `src/i18n/index.ts`, `src/i18n/zh.ts`, `src/i18n/en.ts`

---

### 2. Multi-language Agent system prompts

**Impact:** Agent core engine  
**Description:** `buildDefaultSystemPrompt` now accepts a `language` parameter to generate system prompts in the configured language. Language setting flows from `openflux.yaml` â†’ `createAgentLoopRunner`.

**Modified:** `gateway/src/agent/loop.ts`, `gateway/src/gateway/standalone.ts`

## Improvements

### 3. Agent Loop history management refactoring

**Impact:** Agent core engine  
**Description:** Major refactoring of Agent Loop conversation history management (557 lines changed):
- **Vision data cleanup**: Auto-clean Base64 image data from old Vision messages to reduce token consumption
- **Tool result compression**: Compress early iteration tool call results, keeping only summaries
- **Goal anchor cleanup**: Remove expired goal anchor messages
- **File Integrity Guard**: Detect LLM-hallucinated non-existent files and warn
- **Completion Guard enhancement**: Improved task-blocked detection with multi-block nudging strategy

**Modified:** `gateway/src/agent/loop.ts`

---

### 4. Frontend UI internationalization

**Impact:** Frontend global  
**Description:** `index.html` fully reworked to i18n-driven (396 lines changed). All hardcoded Chinese text replaced with `data-i18n` attributes. Settings panel, chat interface, sidebar, scheduler, and all UI components adapted for bilingual support.

**Modified:** `index.html`, `src/main.ts`, `src/styles/main.css`

---

### 5. Tool descriptions internationalization

**Impact:** Tool system  
**Description:** All built-in tools (browser, desktop, filesystem, email, MCP, memory, notify, office, code execution, process, web search/fetch, Windows control, workflow) â€” descriptions and parameter docs now bilingual.

**Modified:** 20+ files under `gateway/src/tools/`

---

### 6. Language config persistence

**Impact:** Configuration system  
**Description:** `server-config.json` now persists the `language` field. User's language preference survives app restarts. Added `language` field to `config/schema.ts`.

**Modified:** `gateway/src/gateway/standalone.ts`, `gateway/src/config/schema.ts`

---

### 7. Voice service initialization resilience

**Impact:** Voice functionality  
**Description:** TTS/STT service initialization failures now only produce warnings instead of blocking startup, improving fault tolerance in environments without voice support.

**Modified:** `gateway/src/main/voice/stt.ts`, `gateway/src/main/voice/tts.ts`

---

### 8. Tauri command layer optimization

**Impact:** Rust backend  
**Description:** Gateway command layer refactored (`commands/gateway.rs`, `lib.rs`), optimizing process management and communication.

**Modified:** `src-tauri/src/commands/gateway.rs`, `src-tauri/src/lib.rs`
