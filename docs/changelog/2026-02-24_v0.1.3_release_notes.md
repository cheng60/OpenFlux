# OpenFlux v0.1.3 Release Notes

**Created:** 2026-02-24  
**Author:** Development Team  
**Status:** Published  

## Overview

OpenFlux v0.1.3 fixes LLM conversation 404 error after setup wizard and Router managed mode not working on first configuration.

---

## Bug Fixes

### 1. Fix LLM conversation returning 404 after setup wizard

**Impact:** Core functionality (conversation unavailable)  
**Trigger:** Switching LLM provider in setup wizard (e.g., Moonshot → DeepSeek) without manually setting `baseUrl`  
**Root cause:** `handleSetupComplete` used conditional assignment for `baseUrl`, leaving stale URL from previous provider  
**Fix:** Prioritize `payload.baseUrl`, then look up provider's default `baseUrl` from config, fallback to `undefined` (use factory built-in defaults)

**Modified:** `gateway/src/gateway/standalone.ts`

---

### 2. Fix Router managed mode unavailable on first setup

**Impact:** Router managed LLM functionality  
**Trigger:** After configuring Router in setup wizard, the "Use Managed Config" toggle doesn't appear or respond until restart  
**Root cause:** Two sub-issues:
- Field mapping error: Wizard sends `appSecret` but doesn't map to `apiKey` field required by `RouterConfig`
- Missing connection call: No `routerBridge.updateConfig()` after saving Router config

**Fix:** Properly construct `RouterConfig` object with correct field mapping and call `routerBridge.updateConfig()` immediately after save

**Modified:** `gateway/src/gateway/standalone.ts`

---

## Upgrade Notes

- Direct overlay installation, no additional configuration needed
- Existing `server-config.json` and `openflux.yaml` configs are not affected
